1. 결론 요약

현재 구성은 100명 동시 사용을 안정적으로 감당합니다. 이전(동기 방식)에는 동시 50명에서 성공률 41.8%, 600명에서 6.5%로 붕괴했지만, 비동기 큐 + 동시성 게이트 도입 후 100명 동시 요청에서 접수·최종 성공 모두 100.0%를 기록했습니다. 실패가 오류 대신 짧은 대기로 흡수됩니다.
  • 최종 성공률 100.0%: 100건 요청이 모두 결과까지 도달 — 폴백/타임아웃/상위 오류 0건.
  • 빠른 접수: POST는 즉시 202 + jobId 반환 (p95 479ms). 사용자는 기다리지 않고 화면이 진행됨.
  • 결과 지연: 결과까지 중앙값 3.6초, p95 4.7초, 최대 10.8초 — 게이트 대기 포함.
  • 동시성 게이트: 프로세스당 동시 5건 × 3 레플리카 ≈ 15건으로 상위 AI Studio 한계(~20)를 넘지 않게 직렬화.
  • 모델: Claude Sonnet 4.6 기준 측정.

4. 이전 대비 (개선 효과)

이전 동기 방식에서는 LLM 호출을 요청 시점에 그대로 처리해 동시 부하가 몰리면 상위 AI Studio가 건별 500 오류를 반환했습니다(50명 41.8%, 600명 6.5% 성공). 비동기 큐 + 동시성 게이트는 동시 호출 수를 상위 한계 이하로 직렬화하고, 초과분을 짧게 대기시켜 오류를 대기로 전환합니다. 그 결과 100명 동시에서 최종 성공률 100.0%를 달성했습니다.

5. 권장 사항

  • 현 구성 유지: 100명 규모 수업은 현재 설정으로 안정적. 추가 조치 불필요.
  • 더 큰 규모 대비: 600명 이상은 결과 지연이 길어질 수 있음 — 레플리카 수 또는 게이트 값(× 레플리카 ≤ ~20)을 상위 한계에 맞춰 조정.
  • 진행 표시 UX: 접수가 즉시(202)이므로 "생성 중…" 진행 표시로 체감 대기를 자연스럽게.
  • 모니터링: 결과까지 p95와 큐 대기 길이를 운영 지표로 추적해 한계 도달 전 선제 확장.
  • 대규모 재측정: 300·600명 동시 시나리오를 심야에 추가 측정해 결과 지연 곡선 확보 권장.

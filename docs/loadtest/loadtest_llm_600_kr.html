동시 50명에서 이미 성공률이 약 42%로 떨어지고, 400명에서는 5% 미만으로 붕괴합니다. 원인은 일일 토큰 한도가 아니라 동시 호출 시 상위 AI Studio API가 반환하는 일반 500 오류입니다.

1. 결론 요약

현재 구성으로는 600명 규모 수업에서 AI 매거진 생성이 정상 동작하지 않습니다. 가장 가벼운 작업(extractKeyword, 80토큰)으로 측정했음에도, 동시 사용자가 늘수록 성공률이 가파르게 하락했습니다. 무부하 단일 호출은 2.8초/정상이지만, 동시성이 올라가면 상위 제공자(AI Studio)가 건별로 500을 반환합니다.
  • 용량 곡선: 50명 41.8% → 200명 15.3% → 400명 4.7% → 600명 6.5% 성공. 동시성이 핵심 병목.
  • 실패 성격: 일일 토큰 한도("토큰 호출량") 메시지는 전혀 없었음(pool_exhausted=0). 대신 aistudio call (LG_BOOK_x): 500 "오류가 발생했습니다. 다시 시도해 주세요." 형태의 일반 오류.
  • 10키 로테이션의 한계: 키를 돌려도 상위 API 자체가 동시 부하에서 500을 내므로 동시성 여유가 생기지 않음.
  • 지연: 부하 시 중앙값 10초, p95 13~19초 (무부하 2.8초 대비 4~7배).

4. 실측 — 실제 사용 시나리오 (전체 세션)

위 3장은 가장 가벼운 작업(extractKeyword)만 반복한 측정입니다. 실제 사용자는 한 세션에서 약 21개의 LLM 작업(그중 무거운 합성 작업 7개, 각 2,200토큰·15~25초)을 순서대로 호출합니다. 아래는 이 전체 세션 흐름을 그대로 재현해 측정한 값입니다.

5. 실패 원인 분석

실패는 웹 서버(nginx/Next.js)나 인증이 아니라 상위 AI Studio API에서 발생했습니다. 동시 호출이 몰리면 개별 키(LG_BOOK_1…10)가 500 "오류가 발생했습니다"를 반환합니다. 이는 일일 한도 소진과 다른, 동시성/순간 처리량 한계입니다. 따라서 키를 더 추가해도 같은 순간 동시 호출이 많으면 동일하게 실패할 가능성이 높습니다.

6. 권장 사항

  • 동시 호출 제한(큐잉): 앱 레벨에서 동시 진행 LLM 호출 수에 상한을 두고 초과분을 큐로 직렬화 — 순간 스탬피드 방지.
  • 재시도(백오프): 일반 500은 일시적 — 지수 백오프 + 지터로 1~2회 재시도.
  • UX 분산: 생성 트리거 단계를 사용자별로 시차를 두어 동시 호출 집중 완화.
  • 오버플로 폴백: 한도 초과 시 Anthropic(claude) 모드로 일부 트래픽 우회 (x-llm-mode=claude).
  • 운영 시간대 재측정: 이번 실행은 운영 시간대였음 — 심야(off-hours) 재측정으로 동시성 한계와 일일 한도를 분리 확인 권장.
측정 한계. 제한 실행(각 단계 20초)·최경량 작업·운영 시간대 조건. closed-loop는 실제 입장보다 가혹하지만, 실제 수업에서도 동일 단계에서 LLM 호출이 군집하므로 방향성은 유효합니다.

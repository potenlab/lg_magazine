1. Executive Summary

The fix was correct and it worked. Growing the light lane from 1 code (GEMINI_FLASH) to 10 (LG_BOOK_GEMINI + 9 more) cut "code exhausted" errors by 77% and lifted task success 28.7%→45.8%. Pre-deploy, all 10 codes verified 10/10 healthy and a 20-concurrent burst hit 20/20. However, when 600 users run the full 21-task session at once (≈12,600 tasks), the bottleneck is no longer code count but concurrency throughput (gate ~15 + LG AI Studio ~20-concurrent account cap). Tasks now wait in the queue instead of erroring, exceeding the 300s deadline (live probe: queue positions 188–337).
Bottleneck shifted — code exhaustion → concurrency throughput. Before: the single light code exhausted instantly → errors. After: codes are plentiful, but the concurrent execution slots (gate ~15, account ~20) can't keep up with 600 concurrent load → queue backlog → timeouts. Adding codes solves quota/exhaustion, not throughput — throughput is set by the gate and the account ceiling.
  • Task success 28.7% → 45.8% (+60% relative): clear improvement.
  • Upstream errors 4,156 → 963 (−77%): exhaustion errors largely eliminated.
  • Failure mode shifted: timeouts 170→1,055 — tasks queue instead of dying, then exceed the deadline.
  • Light TTR 93.5→236.0s (median): fail-fast errors gone; tasks now wait in a deep queue → number rises.
  • Heavy (Sonnet) TTR: median 296.6s, p95 302.0s — heavy lane also saturated by the concurrency limit.
  • 0/600 sessions completed: 600 concurrent full sessions exceed the current throughput ceiling.

4. Interpretation

Adding codes was the correct first fix (it removed exhaustion errors). But 600 simultaneous full sessions push ≈12,600 tasks at once — an extreme load where LG AI Studio's ~20-concurrent account cap and the local gate (~15) are the throughput ceiling. Real usage doesn't start 600 sessions at the exact same instant; arrivals spread out, so real-world load is far gentler than this test. The async queue still absorbs bursts as waiting rather than hard failure.

5. Recommendations

  • Done (verified): light codes 1→10 — exhaustion errors −77%. Live in production.
  • Confirm the concurrency cap (next): check whether LG AI Studio's ~20-concurrent limit is shared across both lanes or per-lane. Negotiate a higher cap if possible.
  • Tune the gate: raise AISTUDIO_(LIGHT_)MAX_CONCURRENCY × replicas up to the account ceiling.
  • Size realistic concurrency: 600 simultaneous full sessions exceed the ceiling. Set the SLA target against a spread arrival model.
  • Separate bug: judgeBranch fails with reading 'letters' (load-independent). Inspect payload/handler.

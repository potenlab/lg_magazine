3. Interpretation

600 users is well within capacity. The server stayed at 99.94% success with warm responses under 263 ms and used only ~10% of the network line. There is significant headroom above 600.
  • Warm experience is fast: once a user's first page is loaded, navigation is ~263 ms at p95.
  • First-visit (cold) load is the heaviest moment: p95 ~20s because the first page pulls images and audio (~15 MB). This is a one-time cost per user, eased by the asset diet already shipped and by staggering entry.
  • Network is not the bottleneck: peak ~104.1 Mbps is a small fraction of 1 Gbps.
  • AI rotation safe: the live 10-key AI Studio pool covers generation load separately; this test confirms the web/serving tier.

4. Recommendations for Thursday (600 people)

  • Stagger entry over 1–2 minutes if possible, to smooth the simultaneous cold-load spike.
  • Stand by at class start: monitor container health and error logs live.
  • No action needed on capacity — 600 is comfortably handled.

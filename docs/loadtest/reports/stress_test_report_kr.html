1. 결론 요약

다음 주 목요일 600명 수업은 운영 서버가 정상적으로 감당할 수 있습니다. 학생들이 수 분에 걸쳐 순차적으로 접속하는 실제 패턴에서는 1,000명에서도 오류 없이 동작했으며, 서버는 1 Gbps 회선의 약 10~14%만 사용했습니다. 600명은 이 범위 안에 충분히 들어옵니다.
  • 실제 입장 패턴(점진 접속): 1,000명 / 실패율 0% / p95 4.2초 — 600명은 여유.
  • 극단 패턴(전원 동시 고정 부하): 부하가 가장 가혹한 폐쇄루프 테스트에서는 200명부터 지연·실패가 발생 — 이는 실제 사람 행동보다 훨씬 과격한 조건이며, "전원이 같은 초에 새로고침"하는 상황의 상한선을 측정한 값입니다.
  • 병목: 네트워크 회선이 아니라 동시 시작 시점의 정적 자산(이미지/폰트) 동시 다운로드가 주요 부담. 이미 자산 경량화를 완료했고, 입장 분산으로 추가 완화 가능.

5. 목요일 수업(600명) 권장 사항

  • 입장 분산: 가능하면 안내를 통해 1~2분에 걸쳐 입장하도록 유도 (동시 시작 순간의 자산 동시 다운로드 완화).
  • 대기(스탠바이): 수업 시작 시각에 운영팀 모니터링 대기 — 서버 상태/오류 로그 실시간 확인.
  • AI 호출 여유: AI Studio 10개 키 로테이션이 운영에 적용되어 있어 토큰 한도 초과 시 자동 전환 (일 ~1,000회 매거진 생성 가능). 600명은 여유 범위.
  • 자산: 이미지/폰트 경량화 적용 완료 — 최초 진입 부담 감소.
